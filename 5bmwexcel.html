A purpose-built Excel workbook and VBA macro suite that compresses what was once a manual, multi-hour process into a single button click. § 03 / Campaign Analysis Engine

AnalyzeCampaignSheets · dynamic header detection · auto chart placement

The core engine iterates from sheet index 6 onward, calling FindHeaderCol to locate "VIN-17", "Session result", and "Campaign-Name" by scanning row 1 headers — not by fixed column numbers. This makes the macro resilient to BMW portal layout changes across data exports.

Each VIN is counted and classified with a case-insensitive UCase comparison. Contribution rate — each campaign's VIN share of the total fleet across all loaded sheets — is computed in a two-pass loop: the first pass totals all VINs, the second processes each campaign. Three formatted rows are written per campaign with custom top/bottom borders and 60 pt row heights.

  • Dynamic header detection — FindHeaderCol scans every cell in row 1, making the macro immune to column reordering in future BMW exports.
  • Contribution rate — each campaign shows its VIN percentage of the entire fleet across all sheets, not just its own data, giving a genuine at-a-glance priority view.
  • Per-campaign pie chart — 340×155 px, inserted at column E of each block. BMW blue (RGB 0, 51, 153) for success; mid-grey for failure; percentage labels outside.
  • Overall RSU chart — a second chart at N3, 340×200 px, aggregates all campaigns: title auto-set to "Overall RSU Success Rate — [n] VINs".

AnalyzeCampaignSheets · core classification loop 
' Two-pass: first count total VINs fleet-wide
For sheetIndex = 6 To ThisWorkbook.Sheets.Count
  Set ws = ThisWorkbook.Sheets(sheetIndex)
  vinCol = FindHeaderCol(ws, "VIN-17")
  If vinCol > 0 Then
    lastDataRow = ws.Cells(ws.Rows.Count, vinCol).End(xlUp).Row
    For i = 2 To lastDataRow
      If Trim(CStr(ws.Cells(i, vinCol).Value)) <> "" Then
        totalVINs = totalVINs + 1
      End If
    Next i
  End If
Next sheetIndex

' Second pass: classify each campaign sheet
For sheetIndex = 6 To ThisWorkbook.Sheets.Count
  Set ws = ThisWorkbook.Sheets(sheetIndex)
  vinCol      = FindHeaderCol(ws, "VIN-17")
  resultCol   = FindHeaderCol(ws, "Session result")
  campaignCol = FindHeaderCol(ws, "Campaign-Name")

  If vinCol = 0 Or resultCol = 0 _
      Or campaignCol = 0 Then GoTo ContinueNextSheet

  For i = 2 To lastDataRow
    If Trim(CStr(ws.Cells(i, vinCol).Value)) <> "" Then
      vinCount = vinCount + 1
      If UCase(Trim(CStr( _
          ws.Cells(i, resultCol).Value))) = "SUCCESS" Then
        successCount = successCount + 1
      Else
        failureCount = failureCount + 1
      End If
    End If
  Next i

  contributionRate = Round((vinCount / totalVINs) * 100, 1)
Next sheetIndex

VBA · UpdateUnsuccessfulVINs · dedup guard + AutoLine lookup + conditional highlight

Every non-Success VIN triggers an exact VIN-17 match against the Customer Database sheet — compiled from BMW AutoLine, the dealer management system. The match returns the customer's name (column J) and phone number (column R).

The phone number is stored as text with a "+" prefix to preserve international dialling codes that would otherwise be dropped by Excel's numeric formatting. Rows where either field is blank are highlighted in pink (RGB 255, 199, 206), giving the service team a clear visual flag to chase the record manually rather than guessing which entries are incomplete.

A checkbox is inserted into column A of each new row, allowing advisors to tick off follow-ups directly in the sheet — no separate CRM or printout needed.

§ 05 / TC Success Rate Dashboard

Formatted table · inline pie charts · overall RSU aggregation

The TC Success Rate sheet is the at-a-glance output layer. Each campaign occupies exactly three rows — a bold header row with campaign name, VIN count, and contribution percentage; a success rate row; and a failure rate row — all at 60 pt height with matching top/bottom borders.

At column E, a 340×155 px pie chart is automatically inserted and positioned against each campaign block, so the team gets an immediate visual read without opening a separate file. Data labels display percentages only, no values, keeping charts uncluttered.

  • Fleet contribution — each campaign header shows its share of total VINs across all loaded campaign sheets, computed in the two-pass fleet loop.
  • Automatic positioning — each chart's Left/Top is calculated from the cell anchor, so no manual drag-and-drop is ever needed.
  • Overall RSU chart — a larger 340×200 px chart at N3 aggregates every campaign into a single fleet-wide success/failure view, with the total VIN count in the title.
  • Bar chart module — CreateBarChartAtN3 renders a clustered-column chart of all campaign success rates at N3 for comparative trending across months.

§ 07 / Complete Dataset & Pivot Integration

UpdateCompleteDataset · append-only · 7-column flat table

The UpdateCompleteDataset macro produces a single clean flat table that accumulates every VIN across all campaign sheets. It is designed as an append-only living record — re-running the macro on a new batch of campaign files adds rows from the next empty row, never overwriting historical data, making it safe to run incrementally across months.

  • Seven columns — VIN, Session Result, Series, Market, Country, Last I-Level in RSA, Campaign Name — covering all dimensions needed for any pivot analysis or Power BI model.
  • Consistent formatting — every inserted cell is center-aligned, vertically aligned, and bordered on all four edges in a single inner For loop, keeping the table visually clean without a separate formatting pass.
  • Incremental append — the macro locates the last non-empty row in column A and starts writing from the next row, so partial campaign loads are safe and reversible.
  • Header guard — any campaign sheet missing one of the seven required column headers is silently skipped via GoTo ContinueNextSheet, preventing partial writes.
